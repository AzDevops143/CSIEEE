Mathematical Baseline

The defense injects noise sampled from a Laplace distribution centered at 0 with scale scale parameter b:

f(η) = (1 / 2b) * exp(-|η| / b)

Unlike Gaussian noise, Laplace distribution has fatter tails, which increases the likelihood of large boundary hops to scramble clustering.

Academic Thesis and Evaluation Gaps

The Side-Channel Threat: SNOOPFINGER

Virtual reality headsets require high-frequency orientation tracking (72 Hz to 120 Hz) to keep the display aligned with human head positions and prevent motion sickness. Security APIs treat IMU orientation (quaternions) as a zero-permission sensor. SNOOPFINGER exploits this by running in background threads, translating these 3D orientations into 2D gaze locations on the virtual keyboard plane. Because humans naturally pause and focus (dwell) on keys they intend to type, the attack identifies temporal-spatial gaze clusters to decode keys, achieving over 70% word reconstruction rates.

The Flaw in the Original Defense

The initial version of CASOM added Independent, Identically Distributed (IID) Noise to each individual sample. While this scatters points visual-wise and breaks simple clustering algorithms, it represents a fatal conceptual flaw under a strong adversary. Since a user dwells on a key for ~15 to 35 frames (at 72 Hz), an adaptive attacker can group points by their temporal dwell windows and calculate the average centroid. By the Law of Large Numbers, independent zero-mean noise averages back to zero:

E[x_{obfuscated}] = E[x_{raw} + Laplace(0, b)] = E[x_{raw}]

This allows the attacker to recover the clean typing coordinates. Similarly, "Correlated Drift" fails because it shifts consecutive points in the same direction, preserving the relative geometry vectors between keys, which the state-of-the-art BackTree attack exploits to rank matching words.

Evaluating Defense Modes Against BackTree

IID Laplace Noise Vulnerable (38%)

Adds noise independently per frame. Attacker dwell averaging cancels out the noise, restoring key positions.

Correlated Drift Vulnerable (29%)

Uses Ornstein-Uhlenbeck drift. Preserves consecutive relative spatial vectors, failing against BackTree.

Block Offset Secured (4%)

Recommended. Draws Laplace noise once per block (15 frames) and holds constant. Displaces keys independently, surviving averages.

Critical Gaps in SNOOPFINGER's Attack Model

While SNOOPFINGER's threat model holds under passive conditions, it contains two fundamental assumptions that can easily be exploited to neutralize the threat:

  • User Deviation Gestures (Gap 1 - Manual Bypass): The attack relies on passive, direct head trajectories matching key coordinates. If a user intentionally makes random or sweeping head movements (e.g. nodding, circular detours) while typing, the background attacker receives these gestures. The attack has no mechanism to differentiate real typing dwells from user-driven detour gestures, resulting in completely scrambled keystroke outputs.
  • External Sensor Tampering & Environmental Noise (Gap 2 - Environmental Fragility): The attack assumes that head orientation data represents only typing gestures. Physical vibrations (like riding a bumpy bus, walking) or hardware-level sensor noise injection alter the raw telemetry. The attack algorithm has no sensor-origin verification and cannot separate target key dwells from external vibrations, rendering the attack fragile outside a static environment.
